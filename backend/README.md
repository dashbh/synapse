# Backend — A2UI Platform

FastAPI backend for Synapse. Implements the RAG query pipeline, document ingestion, session management, and full LGTM observability.

**Status:** MVP Complete — core RAG pipeline, session persistence, ingestion, and telemetry all operational.

---

## Quick Start

```bash
cd backend
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # fill in your keys
python main.py         # starts on http://localhost:8000
```

Or via Docker (recommended):

```bash
# From repo root
make dev     # hot-reload (uvicorn --reload)
make prod    # production (uvicorn workers)
```

## Endpoints

| Method | Path | Description |
|---|---|---|
| `POST` | `/api/agents/knowledge-qa` | RAG query → SSE stream of A2UI messages |
| `POST` | `/api/agents/ingest` | Document ingestion (PDF/DOCX/TXT/MD) → SSE progress stream |
| `GET` | `/api/sessions` | List all sessions (id, name, message_count, dates) |
| `GET` | `/api/sessions/current` | Return active session from cookie, or 404 |
| `POST` | `/api/sessions` | Create new session (UUID), set 30-day `kqa_session_id` cookie |
| `PATCH` | `/api/sessions/{id}` | Rename a session — body: `{ name: string }` |
| `POST` | `/api/sessions/{id}/activate` | Switch active session (updates cookie) |
| `GET` | `/api/sessions/{id}/messages` | Return pre-paired turns `[{ query, a2ui_payload }]` for hydration |
| `DELETE` | `/api/sessions/{id}` | Delete session and all its messages |
| `GET` | `/health` | Health check — verifies OpenAI API key + Supabase connectivity |

### Query params for `/api/agents/knowledge-qa`

- `query` (required) — natural language question
- `surface_id` (required) — per-turn UUID generated by FE (`qa-turn-<uuid>`); echoed in both A2UI messages
- `session_id` (optional) — UUID; if provided, last 10 messages prepended as history

## A2UI Message Sequence

```
Message 1  createSurface     → creates empty surface (immediate, T+0ms)
Message 2  updateComponents  → Architect's Triad + source list (after RAG completes, T+1–30s)
```

Stream closes after Message 2.

---

## RAG Pipeline

```
embed_query     → OpenAI text-embedding-ada-002 (1536-dim)
retrieval       → Supabase pgvector (match_document_chunks RPC, MIN_SIMILARITY=0.78)
llm_completion  → OpenAI gpt-4o-mini (Architect's Triad SYSTEM_PROMPT + last-10-msg history)
stream_response → SSE yield createSurface + updateComponents
store_messages  → async background task (user query + A2UI payload stored)
```

Each step is instrumented with an OTel span — visible in Grafana/Tempo.

## Observability

The backend ships with full LGTM instrumentation — see `app/telemetry/`.

- **Traces:** Every request gets an HTTP span; RAG pipeline emits 4 child spans (`embed_query`, `retrieval`, `llm_completion`, `stream_response`)
- **Logs:** structlog JSON — every line carries `trace_id`, `span_id`, `session_id`, `request_id`
- **Metrics:** Prometheus histogram `synapse_rag_step_duration_seconds` per RAG step
- **Propagation:** W3C traceparent header extracted from FE; `X-Trace-ID` returned in response

Start the LGTM stack alongside the app:

```bash
make dev   # includes observability sidecar by default
```

Grafana: http://localhost:3001 (anonymous admin, datasources auto-provisioned)
